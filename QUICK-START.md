### LLBLGen Pro
- Use **Adapter pattern** (not Self-Servicing)
- Coordinate with Database Engineer on schema changes
- Repository implementations are in **Core project**, not Data

### DTO Mapping
- Use **AutoMapper** to map between LLBLGen entities and DTOs
- Input DTOs: `CreateAreaInput`, `UpdateAreaInput`
- Output DTOs: `AreaOutput`, `AreaListOutput`

### Migration Naming
- Format: `VXXX_DescriptiveName.cs`
- Example: `V1_AddInitialTables.cs`, `V2_AddProductCategories.cs`
